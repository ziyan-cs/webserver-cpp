## 2026-05-16

### 实现进度
1. 实现 TcpServer 核心逻辑，追加了 `accept/recv/send IO` 流程
2. 添加 `main` 入口和终端交互逻辑
3. v0.1 版本功能全部正常运行
4. 新增 `SIGINT(Ctrl+C)` 信号处理，通过全局标志位 `g_running` 优雅退出运行

### 遇到问题
1. 输出错误：`LOG_INFO` 打印客户端发来消息乱码
    - 现象：直接用 `<< buffer.data()` 输出，导致乱码和越界
    - 根因：`recv()` 收到的数据是二进制流，不会自动加上 `\0` 结尾
    - 解决：`recv()` 前置 `buffer[n] = '\0'`
2. 服务器无法响应服务端 `Ctrl+C` 信号
    - 现象：发送 `SIGINT` 信号后，服务器仍卡在 `accept()/recv()` 的阻塞调用中，无法退出
    - 根因：阻塞系统调用被信号打断时未处理 EINTR 错误，且循环条件未响应信号标志位
    - 解决：`accept()` 和 `recv()` 均添加 `errno == EINTR && !g_running` 判断以退出循环
